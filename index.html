Magical REM

Responsive Font Sizing

You're doing it wrong

Font Sizing

two techniques

  • Size with px
  • Size with em

Sizing with px

The reliable and consistent way

.some-div {
  font-size: 30px;
}
@media (max-width: 1000px) {
  .some-div { font-size: 22px; }
}
@media (max-width: 500px) {
  .some-div { font-size: 13px; }
}

The Problem?

Imagine 50 different font-size declarations in px within your stylesheet

That's 50 fail points

Sizing with em

Relative to its parent font size.


body { font-size: 62.5%; }
h1 { font-size: 2.4em; } /* =24px */
p  { font-size: 1.4em; } /* =14px */

The Problem?

Font size compounds!

A div within a 14em div isn't 14px, it's 20px. Go another level deeper and it's 27px!

Sizing with rem

Relative to the root - or the html - element.

Define a single font size on the html element and define all rem units to be a percentage of that.


html { font-size: 62.5%; }
body { font-size: 1.4rem; } /* =14px */
h1   { font-size: 2.4rem; } /* =24px */

The best part of REM?

Elegant cascading using media queries on only one element

html { font-size: 62.5%; }
@media (max-width: 1000px) {
  html { font-size: 55%; }
}
@media (max-width: 500px) {
  html { font-size: 48%; }
}

But what about browser support?

You might be surprised...

Gotchas

  • No IE 8 or Opera Mini support (Modernizr to the rescue!)
  • IE 9 & IE 10 do not support rem units when used in the font shorthand property
  • IE 9, 10 and 11 do not support rem units when used in the "line-height" property when used on :before and :after pseudo elements
  • Borders sized in "rem" disappear when the page is zoomed out in Chrome.

Questions? Keep in mind...

  • Skeleton.css uses rem by default
  • Bootstrap V4 will be moving to rem units*
